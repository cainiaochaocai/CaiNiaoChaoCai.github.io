想写点SpringMVC的入门

SpringMVC 是一个表现层框架,是Spring框架的一部分

SpringMVC框架流程:

1.客户端发送请求到 前端控制器;
2.前端控制器 接收请求后调用 处理器映射器 查询Handler.
3.处理器映射器 返回处理器执行链.
4.前端控制器向 处理器适配器 请求执行Handler
5.处理器适配器 执行 Handler 并返回ModleAndView
6.前端控制器向 视图解析器 请求解析视图
7.视图解析器返回view
8.前端控制器渲染视图并返回给客户端.

组件说明:

1.前端控制器:DispatcherServlet ,会拦截客户端的请求,是SpringMVC的入口.需要在web.xml中配置该Servlet,同时配置拦截路径,拦截所有请求用的是"/",而不是"/*",同时还要配置SpringMVC.xml配置文件.
2.处理器映射器:HandlerMapping ,可以根据不同的URL找到相对应的Handler,可以有不同的配置方式:配置文件,实现接口,注解,一般用注解的方式.
3.处理器适配器: HandlerAdapter ,执行不同的Handler.
    注意点:默认的处理器映射器和处理器适配器已经过时,可以在SpringMVC.xml中用<mvc:annnotation-driven /> 代替.
4.视图解析器: InternalResourceViewResolver,支持jsp是解析,可以在配置文件中配置该bean,同时配置前缀(prefix)和后缀(suffix).
5.Handler和View要自己书写.

Handler的编写:

1.编写一个类,打上@Controller注解(前提是在配置文件中开启注解扫描);
2.编写方法,打上@RequestMapping("xxx")注解,里面的值对应不同的请求路径;
    方法的返回值有三种:
        a.返回ModelAndView对象,在方法中new出来,添加数据和指定view(底层是将数据设置到request域中): 添加数据:addObject("xxx",xxx);指定视图:setViewName("xxx");
        b.返回void,一般在异步请求时使用.
        c.返回String字符串:1.视图的名称;2.重定向;3.请求转发.

    方法的参数:
        1.默认支持的参数类型有:
            HttpServletRequest
            HttpServletResponse
            HttpSession
            Model/ModelMap:后面是前面的实现,用这两种做参数就不需要返回ModelAndView对象了,直接用model.addAttribute("xxx",xxx)设置参数,然后返回视图的字符串就行.
